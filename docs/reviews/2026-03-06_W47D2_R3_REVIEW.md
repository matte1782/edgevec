## Correctness Deep-Check: percentile() Function

Verified the linear interpolation implementation against edge cases:

| Input | p | Expected | Actual (by code trace) |
|:------|:--|:---------|:-----------------------|
| `[]` | 50 | 0 | 0 (line 72 early return) |
| `[7]` | 99 | 7 | 7 (line 73 early return) |
| `[1, 2, 3, 4, 5]` | 50 | 3 | rank=2.0, lo=2, hi=2, frac=0 -> sorted[2]=3 |
| `[1, 2, 3, 4, 5]` | 99 | 4.96 | rank=3.96, lo=3, hi=4, frac=0.96 -> 4 + 0.96*(5-4)=4.96 |
| `[10, 20, 30]` | 0 | 10 | rank=0, lo=0, hi=0, frac=0 -> sorted[0]=10 |
| `[10, 20, 30]` | 100 | 30 | rank=2.0, lo=2, hi=2, frac=0 -> sorted[2]=30 |

All correct. The interpolation handles boundary cases (p=0, p=100) without index overflow because `Math.ceil(rank) == Math.floor(rank)` when rank is integer, so `sorted[lo] == sorted[hi]`.

## Attack Vector Results

### 1. RAYON CORRECTNESS (product.rs)

- **Per-subspace seeding:** `ChaCha8Rng::seed_from_u64(42 + m as u64)` used in `train_subspace()` helper (line 344). Same function called from both `#[cfg(feature = "parallel")]` and `#[cfg(not(feature = "parallel"))]` paths (lines 466-474). PASS.
- **Determinism:** `train_subspace()` is a pure function -- takes immutable `&[&[f32]]`, creates its own RNG, returns `Vec<f32>`. No shared mutable state. Parallel execution via `into_par_iter().map()` is embarrassingly parallel. PASS.
- **Feature gating:** `#[cfg(feature = "parallel")]` on import (line 43) and both code paths (lines 465, 471). No compile-time leakage. PASS.
- **Helper function:** `train_subspace()` (lines 330-353) is a standalone function that extracts subvectors, creates per-subspace RNG, calls `kmeans()`, and returns centroids. Clean extraction. PASS.
- **No `unwrap()` in new code:** `train_subspace()` calls `kmeans()` which returns `Vec<f32>` (not Result). No `unwrap()` added. PASS.
- **Determinism test:** `test_parallel_train_deterministic` (line 2038) trains two identical codebooks and compares centroids + encoded vectors. This test runs under both `--features parallel` and default, proving equivalence. PASS.
- **Convergence threshold:** `train_subspace()` correctly passes `convergence_threshold` through to `kmeans()` (line 336, 346). PASS.

**Deviation from plan:** Plan W47.4a says "gated with `#[cfg(not(target_arch = "wasm32"))]`" but actual implementation uses `#[cfg(feature = "parallel")]`. The feature-flag approach is SUPERIOR: it gives users explicit opt-in, and WASM builds naturally exclude rayon because `parallel` feature is not enabled for WASM targets. The `cfg(target_arch)` approach would silently enable rayon on all non-WASM targets without user consent. ACCEPTED as improvement over plan.

**G3 FAIL analysis:** The recall numbers (0.39/0.53) are significantly below 0.90.
